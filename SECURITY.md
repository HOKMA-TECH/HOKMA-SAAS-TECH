# Security Policy

## Supported Versions

This repository is currently in active development. Security fixes are applied to the `main` branch.

## Reporting a Vulnerability

Please do not open public issues for sensitive vulnerabilities.

Use one of the following:

1. GitHub Security Advisories (preferred)
2. Private contact with maintainers

When reporting, include:

- Affected component/file
- Impact and severity estimate
- Reproduction steps or proof of concept
- Suggested mitigation (if available)

## Security Principles Used in This Project

- Frontend and desktop client are treated as untrusted.
- Authorization is enforced in backend/database policies.
- Least privilege by default.
- Sensitive data is private by default.
- Multi-tenant isolation is mandatory.
